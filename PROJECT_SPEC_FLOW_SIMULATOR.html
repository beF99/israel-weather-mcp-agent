תרשים פרויקט + סימולציה מפורטת של המטלה

בדף הזה יש שתי תצוגות נפרדות: `messages (היסטוריה חיה)` עבור תוכן המערך בפועל, ו-`שלבי ביצוע` עבור יומן השלבים שבוצעו. לחיצה על כל שלב או אובייקט פותחת חלון מפורט עם הסבר וקוד.

תרשים זרימה ראשי

1) התחלת Query
יצירת `messages` ראשוני עם הודעת משתמש.
↓
2) קריאה למודל עם tools
`chat.completions.create(messages, tools)`
↓
3) בדיקה אם יש tool_calls
אם אין: תשובה סופית. אם יש: ממשיכים לכלי.
↓
4) הוספת assistant להיסטוריה
שומרים גם את tool_calls במבנה ההודעה.
↓
5) הפעלת כלי MCP
open / enter / select / extract לפי החלטת המודל.
↓
6) הוספת role=tool להיסטוריה
שומרים את פלט הכלי כחלק מההקשר.
↓
7) לולאה לסבב נוסף
שולחים שוב את כל `messages` המעודכן.
↓
8) תשובה סופית למשתמש
רק כשאין יותר `tool_calls`.

כלים שנייצר ב-MCP Israel

open_weather_forecast_israel
פותח דפדפן ונכנס לאתר התחזית.
enter_weather_forecast_city_israel
מקליד שם עיר בשדה החיפוש.
select_weather_forecast_city_israel
בוחר עיר מהרשימה הנפתחת.
extract_weather_forecast_content_israel
מחלץ טקסט תחזית לדיאלוג עם המודל.
בחרי תרחיש כדי להתחיל.
ב"תרחיש ב" תראי ריצה מלאה: open -> enter -> select -> extract, ואז תשובה סופית.

messages (היסטוריה חיה)

כאן מוצג תוכן המערך בפועל. לחצי על אובייקט כדי לראות JSON גולמי כמו ב-F12.

התשובה הסופית למשתמש

עדיין אין תשובה סופית.

שלבי ביצוע (יומן היסטורי)

כל שלב נשמר כאן עם שם + תיאור קצר. לחיצה על שלב פותחת הסבר מורחב ודוגמת קוד.
